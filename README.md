# GraphExplorer

Сучасний інтерактивний інструмент для візуалізації математичних функцій, побудований на архітектурних принципах **SOLID** та **GoF Design Patterns**.

##  Архітектура та патерни (Design Patterns)

Проєкт демонструє практичне застосування 11 патернів проєктування для створення гнучкої та масштабованої системи.

###  Породжувальні патерни (Creational)
* **Singleton (`SingletonSettings`)**: Гарантує єдину точку доступу до глобальних налаштувань застосунку (тема, конфігурація масштабу).
* **Builder (`GraphConfigBuilder`)**: Забезпечує покрокове створення об'єктів конфігурації графіка, дозволяючи легко налаштовувати колір, товщину та формулу.

###  Структурні патерни (Structural)
* **Adapter (`MathAdapter`)**: Адаптує інтерфейс бібліотеки `math.js` до потреб нашого застосунку, включаючи вбудовану валідацію введених формул.
* **Proxy (`MathCompilerProxy`)**: Проксі-об'єкт, який кешує скомпільовані функції, щоб уникнути повторних високонавантажених обчислень при зміні стану UI.
* **Facade (`GraphExplorerFacade`)**: Надає спрощений інтерфейс для взаємодії з підсистемами рендерингу, розрахунків та історії.

###  Поведінкові патерни (Behavioral)
* **Strategy (`RenderStrategy` / `ColorStrategy`)**: Використовується для вибору алгоритму малювання кривих та динамічної зміни палітри кольорів залежно від обраної теми (Light/Dark).
* **Command (`AddGraphCommand`)**: Інкапсулює дію додавання графіка в об'єкт, що дозволяє зберігати історію дій та реалізовувати функцію скасування (**Undo**).
* **Observer (React Event System)**: Відстежує стан курсора на Canvas, забезпечуючи реактивне оновлення координат точок у реальному часі.
* **Template Method (`BaseRenderer`)**: Визначає базовий алгоритм побудови координатної площини (сітка -> осі -> дані), залишаючи деталізацію малювання самим підкласам.
* **Memento (`History`)**: Зберігає знімки (snapshots) стану списку графіків для можливості швидкого відновлення попереднього стану системи.
* **Iterator**: Використовується для послідовного перебору та рендерингу колекції графічних об'єктів без розкриття їхньої внутрішньої структури.

##  Основні можливості
- **Динамічний рендеринг**: Побудова графіків будь-якої складності за допомогою `math.js`.
- **Багатоколірність**: Кожен новий графік автоматично отримує унікальний колір із палітри.
- **Інтерактивність**: Наведення курсора на графік відображає точні координати $(x, y)$.
- **Валідація**: Система `hot-toast` сповіщень про помилки в синтаксисі формул.
